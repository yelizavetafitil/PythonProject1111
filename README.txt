РУП БЕЛНИПИЭНЕРГОПРОМ
==========

Описание
--------
РУП «Белнипиэнергопром» — веб-портал на Flask для публикации внутренних ссылок и управления доступом по LDAP и группам.

Что делает проект
-----------------
- Авторизация через LDAP (Active Directory)
- Отображение ресурсов по категориям
- Ограничение видимости ресурсов по группам
- Админ-раздел для управления ресурсами и группами

Как поднять проект с нуля
-------------------------
1) Склонируйте репозиторий:
   git clone https://github.com/yelizavetafitil/PythonProject1111.git
   cd PythonProject1111

2) Создайте виртуальное окружение:
   Windows PowerShell:
   py -m venv .venv
   .\.venv\Scripts\Activate.ps1

   Linux/macOS:
   python3 -m venv .venv
   source .venv/bin/activate

3) Установите зависимости:
   pip install --upgrade pip
   pip install -r requirements.txt

4) Запустите приложение:
   python app.py

5) Откройте в браузере:
   http://127.0.0.1:5004

Настройка LDAP
--------------
Перед запуском проверьте LDAP-настройки в app.py, блок LDAP_CONFIG:
- uri
- base
- bind_dn
- bind_password
- user_attr

Важно
-----
- Папка .venv не передается в git.
- Окружение создается локально на каждой машине.
- Все библиотеки ставятся из requirements.txt.

Модуль "Табель" в едином сервисе
--------------------------------
Код табеля перенесен из отдельного проекта в этот Flask-сервис.
Основные маршруты:
- /tabel
- /api/tabel/meta
- /api/tabel/leaders
- /api/tabel/search-fio
- /api/tabel/status-month

Источник данных и кэш:
- TABEL_BASE_DIR (по умолчанию \\srv-doc\ТАБЕЛЬ — на Linux и Windows одинаково в конфиге)
- TABEL_LEADERS_FILE (по умолчанию \\srv-doc\ТАБЕЛЬ\ОЦ\Список руководителей.xlsx)
- tabel_fs.py — на Linux: кодировки имён (utf-8/cp1251) и чтение UNC через mount /mnt/tabel без смены путей в настройках
- TABEL_SHEET_NAME (по умолчанию: Табель_3)
- TABEL_CACHE_FILE (по умолчанию: tabel_portal_cache.json)

Доступ к сетевой папке (важно для админов):
- На Windows по умолчанию перед чтением табелей выполняется net use под учёткой OC1
  (как в старом сервисе табеля): TABEL_SMB_USER=oc1@local.energoprom.by,
  TABEL_SMB_PASSWORD — из переменной окружения или тот же пароль, что LDAP bind_password.
- TABEL_SMB_ENABLED=1 (по умолчанию на Windows) / 0 — отключить и читать под учёткой процесса.
- TABEL_SMB_NET_USE_TIMEOUT=20 — таймаут net use (сек), если шара недоступна.
- Страница /tabel открывается сразу из локального кэша; сканирование \\\\srv-doc идёт в фоне.
- Логин пользователя на портале (rapeiko, fitil и т.д.) к \\srv-doc не подставляется.
- На продакшене служба может работать под другой AD-учёткой — тогда задайте TABEL_SMB_USER
  и TABEL_SMB_PASSWORD для учётки с правами на \\srv-doc\ТАБЕЛЬ (раньше использовали svc-info).

Для продакшна как единого сервиса:
1) Пользователи работают только через этот проект (/tabel).
   Доступ к любому ресурсу на главной и к его странице: группа на карточке ресурса (скрин «Доступ для групп»)
   ИЛИ соответствующий список в /manage (табель, телефон, бронирование, ИИ и т.д.) — достаточно одного условия.
   Мастер-админы — всегда. Блок «Расширенный доступ к ресурсам» — только право редактировать плитки на главной.

Группа «Пользователи» (доступ к ресурсам)
-------------------------------------------
При первом запуске создаётся группа «Пользователи»; если в ней нет участников — подтягиваются
учётные записи из AD. При каждом входе пользователь добавляется в эту группу. Ресурсы, привязанные
к группе «Пользователи», видят все авторизованные пользователи. На /manage мастер-админ может
нажать «Синхронизировать Пользователи из AD» для повторного обновления списка.
2) Старый сервис табеля (PythonProject15) после приемки отключается.
3) Прямой доступ к старому сервису/порту закрывается firewall-правилом.

Экспорт / импорт разграничений доступа
---------------------------------------
На странице /manage (только мастер-админы) доступны кнопки выгрузки и загрузки JSON
со всеми группами, участниками групп, разделами, ресурсами и связями «ресурс — группы»,
а также списками расширенного доступа (телефонная книга, табель, бронирование, админка ресурсов, ИИ).
Имя файла при выгрузке: access_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.json.
Импорт полностью заменяет указанные связи и списки; перед импортом показывается модальное окно подтверждения.

Выгрузка / загрузка полной базы (SQLite)
-----------------------------------------
Там же: скачать файл database_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.db (согласованная копия через SQLite backup)
и загрузить .db с проверкой целостности. Перед заменой текущая database.db копируется в
database_before_import_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.db в каталоге приложения. На продакшене при
нескольких воркерах Gunicorn возможны блокировки файла — на время замены лучше остановить
процессы или использовать один воркер.

Размещение на сервере (продакшен)
-----------------------------------
На боевом сервере не используйте встроенный сервер Flask в режиме отладки.
Рекомендуется Gunicorn за reverse proxy (nginx) или контейнер Docker.

1) Установка зависимостей и переменных окружения:
   export FLASK_DEBUG=0
   export PORT=5004

2) Запуск через Gunicorn (слушает 127.0.0.1:5004, см. deploy/gunicorn.conf.py):
   gunicorn -c deploy/gunicorn.conf.py wsgi:app

Курсы НБ РБ на главной (api.nbrb.by)
------------------------------------
Локально обычно работает сразу. На сервере за корпоративным прокси/МЭ возможна ошибка
SSL (self-signed certificate in certificate chain) — трафик перехватывается своим
сертификатом. Варианты для администратора:

  export NBRB_SSL_CA_FILE=/path/to/corporate-root-ca.pem
  (предпочтительно: корневой сертификат вашей организации)

  export NBRB_SSL_VERIFY=0
  (временно отключить проверку SSL только для запросов к НБ РБ; менее безопасно)

После изменения переменных перезапустите Gunicorn/systemd.

3) Nginx: скопируйте пример deploy/nginx-site.conf.example и настройте server_name,
   SSL и proxy_pass на upstream flask_app.

4) systemd: пример юнита — deploy/portal.service.example (поправьте User и пути).

5) Docker на сервере:
   docker compose build && docker compose up -d
   При необходимости добавьте в docker-compose.yml том для каталога «БАЗА ЗНАНИЙ»
   и сетевых путей табеля через переменные окружения.

При SQLite рекомендуется workers=1 в Gunicorn (см. deploy/gunicorn.conf.py).

Запуск на сервере, чтобы читался табель (\\srv-doc)
---------------------------------------------------
Главное: папки читает УЧЁТКА ПРОЦЕССА на сервере, не пользователь с сайта.

Шаг 1 — сервер и учётка
  - Портал ставится на машину в сети офиса (доступ к srv-doc без VPN с ноутбука).
  - Создайте (или используйте) AD-учётку службы, например ENERGOPROM\svc-portal
    (можно существующую OC1, если так решит админ).
  - Админ srv-doc выдаёт этой учётке Read на:
      \\srv-doc\ТАБЕЛЬ
      \\srv-doc\ТАБЕЛЬ\ОЦ\Список руководителей.xlsx

Шаг 2 — проверка ДО запуска портала
  Войти на сервер под этой учёткой (или «Запуск от имени…» в cmd) и выполнить:
    dir \\srv-doc\ТАБЕЛЬ
  Если ошибка — портал тоже не прочитает; сначала права/сеть, потом Flask.

Шаг 3 — Windows Server (типовой вариант для UNC)
  1) Клонировать проект, venv, pip install -r requirements.txt
  2) Переменные (системные или в службе):
       FLASK_DEBUG=0
       PORT=5004
       TABEL_BASE_DIR=\\srv-doc\ТАБЕЛЬ
       TABEL_LEADERS_FILE=\\srv-doc\ТАБЕЛЬ\ОЦ\Список руководителей.xlsx
  3) Запускать НЕ под личным профилем разработчика, а как службу Windows
     под AD-учёткой из шага 1 (см. deploy/windows-service-nssm.example.txt).
     На Windows используйте waitress (Gunicorn — только Linux):
       pip install waitress
       waitress-serve --listen=0.0.0.0:5004 wsgi:app
     Рабочая папка: каталог проекта (где database.db и tabel_portal_cache.json).
  4) Слушатель: порт 5004 или reverse proxy (IIS/nginx) на 127.0.0.1:5004
  5) На Linux с Gunicorn: смонтировать SMB (cifs) в /mnt/tabel и задать
       TABEL_BASE_DIR=/mnt/tabel
     Пользователь systemd User= должен иметь доступ к точке монтирования.

Шаг 4 — после запуска
  - Открыть /tabel и проверить поиск сотрудника; при первом запуске дождаться фонового сканирования.
  - Первое сканирование может занять 1–2 минуты; кэш: tabel_portal_cache.json в каталоге проекта.

Шаг 5 — пользователи портала
  - Директорам/сотрудникам права на \\srv-doc НЕ нужны — только вход на сайт и ссылка /tabel.
